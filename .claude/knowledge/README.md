# .claude/knowledge/ — 프로젝트 지식 저장소

Git committed → 모든 워크트리/브랜치에서 공유됨.

## 파일 구조

| 파일 | 용도 | 언제 기록 |
|------|------|-----------|
| `expert-tips.md` | X/웹 고수 팁 (출처 포함) | 외부에서 유용한 정보 발견 시 |
| `til.md` | Today I Learned — 문제→해결 쌍 | 삽질 끝에 해결했을 때 |
| `patterns.md` | 프로젝트 고유 패턴/관례 | 새 패턴 발견/확립 시 |
| `decisions.md` | 아키텍처 결정 기록 (ADR-light) | 중요한 기술 결정 시 |

## 역할 분리

- **글로벌 memory/** (`~/.claude/projects/.../memory/`): 사용자 개인 선호, 크로스 프로젝트
- **레포 knowledge/** (이 폴더): 이 프로젝트 고유 지식, git으로 공유
- **SCRATCHPAD.md**: 세션 연속성 (현재 진행 중인 작업, 휘발성)

## 기록 원칙

1. 코드에서 읽을 수 있는 것은 기록하지 않음 (파일 구조, import 관계 등)
2. **"왜"를 기록** — "무엇"은 코드가, "왜"는 knowledge가 담당
3. 최신 유지 — 더 이상 맞지 않는 항목은 삭제하거나 업데이트
