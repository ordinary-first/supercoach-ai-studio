# /brainstorm — 아이디어 탐색 & 설계 문서 생성

새로운 기능이나 개선 아이디어를 탐색하고, 설계 문서 + 수락 기준 초안을 생성합니다.
구현은 하지 않습니다. 설계만.

## 주제

$ARGUMENTS

## 선행 조건 (Knowledge-First)

탐색 시작 전:
1. `.claude/knowledge/decisions.md` — 관련 기존 결정 확인
2. `.claude/knowledge/patterns.md` — 기존 패턴과 충돌 여부 확인

## 실행 흐름

### Round 1: 문제 정의
1. "이 기능이 해결하려는 **핵심 문제**는 무엇인가요?"
2. "이 기능의 **주요 사용자**는 누구인가요? (신규/기존/파워유저)"
3. "이 기능이 **없으면** 어떤 불편함이 있나요?"

### Round 2: 범위 & 제약
1. "이 기능에 **반드시 포함해야 할** 핵심 동작 3가지는?"
2. "이 기능에서 **제외할** 것은? (MVP 범위)"
3. "기존 기능(목표/할일/피드백/시각화/코치)과 **어떻게 연결**되나요?"

### Round 3: 기술적 탐색
프로젝트 코드를 탐색하여:
1. "기존 {관련 컴포넌트}를 재사용/확장할까요, 새로 만들까요?"
2. "데이터는 Firestore에 어떤 구조로 저장할까요?"
3. "AI 코치와 연동이 필요한가요?"

### 설계 문서 생성

`docs/plans/{topic-kebab-case}.md` 생성:

포함할 섹션: 배경 & 문제, 목표, 범위(포함/제외), 아키텍처(생성/수정 파일),
데이터 모델, UI 흐름, 기존 기능 연결점, 수락 기준 초안.

### 완료

```
설계 문서 생성됨: docs/plans/{name}.md
다음 단계: /build {feature 설명} 으로 구현 시작
```
