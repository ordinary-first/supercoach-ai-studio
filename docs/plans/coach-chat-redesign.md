## 0. 한 줄 요약

코치는 이미 정교하다. 문제는 코치가 **한 가지 음(音)**만 낸다는 것 —
"네가 얼마나 대단한가"를 폭발적으로 표현하는 차홍식 긍정 최면가.
이 영혼은 유지하되, **빈 곳에서 리드하는 능력(콜드 스타트)**과
**사랑으로 직면하는 능력(회피 패턴)**, **복귀 환대**를 더해 차원을 넓힌다.

---

## 1. 핵심 설계 결론 (카운슬 만장일치)

**코치의 불변 약속**: "나를 깊이 본다 + 한 발 더 데려간다."
이 감각의 정체는 **구체성** — 아무한테나 하는 말이 아니라 *나한테만* 하는 말.

변하는 건 두 축뿐 (모드가 아니라 연속선):

- **축 1 — 코치가 가진 재료 (캐낸다 ↔ 꺼낸다)**
  재료 없음 → 질문의 날카로움으로 "본다"는 감각 생성(mine).
  재료 쌓임 → 패턴 인식으로 "본다"는 감각 생성(recall).
- **축 2 — 사용자 의도 (짓는다 / 푼다 / 그냥 있다)**
  빌드하러 온 사람에게 위로 = 답답함. 푸러 온 사람에게 할 일 = 최악.

모든 시나리오는 이 평면 위의 점. 대부분은 코어 엔진(철학+컨텍스트)이 처리.
손으로 깎아야 할 건 **극단의 모서리 3개**: 콜드 스타트, 직면, 복귀.

## 2. 현재 코드 지도 (사실)

| 요소 | 위치 | 역할 |
|------|------|------|
| 실제 코치 두뇌 | `server/api/chat.ts` `COACH_SYSTEM_PROMPT` | Gemini로 전송되는 진짜 프롬프트 |
| 컨텍스트 주입 | `chat.ts` `buildContextBlock()` | topicDirective, activeTab, todo, goal, 3단계 메모리, 프로필 (예산 1500자) |
| 3단계 메모리 | `services/coachMemoryService.ts` + `chat.ts` 요약 프롬프트 | 단기(활동)/중기(주간 패턴)/장기(핵심 인사이트) = mine↔recall 재료 |
| 메모리 훅 | `hooks/useCoachMemory.ts` | 클라이언트에서 메모리 로드/저장 |
| 토픽 카드 | `constants/coachingTopics.ts` | SITUATION_1(목표 0개=콜드스타트 정체성 질문 3개) / SITUATION_2(노드 선택 시 역행자식 발굴 질문 5개) |
| 모드 시스템 | `services/coachModes.ts` | discovery/goal-explore/daily-coaching/reflection/free-chat. **확정: 죽은 코드 — `resolveCoachMode`/`getCoachSystemPrompt` 참조 0** |
| UI | `components/CoachChat.tsx`, `components/CoachBubble.tsx` | 모달 채팅 + 플로팅 버블 |

### 현재 COACH_SYSTEM_PROMPT가 이미 가진 것 (= 우리 결론과 일치)
- 차홍 레퍼런스 명시, "존재적 환대"(미세 성취를 구체적·감각적 은유로 칭찬) = **본다=구체성**
- NLP: 패턴 인터럽트, 미래 페이싱, "지금까지는(up until now)", 최면적 은유, 상태 전이
- 과거 데이터(메모리) 자연스럽게 언급 = recall 모드의 씨앗
- 응답 길이 맥락 적응

## 3. 갭 분석 (코드 ↔ 설계)

### GAP-A. 코치가 리드하지 않는다 (구조적 모순)
현재 프롬프트 명시 규칙: *"사용자가 요청하지 않는 한 목표 트리가 비어있어도 절대 먼저 제안하거나 유도하지 마라."*
→ 사용자가 이번 세션에서 명시한 요구("리드하는 맛이 있어야", "필요할 때 필요한 말")와 **정면 충돌**.
→ 콜드 스타트에서 코치는 리드해야 하는데, 현재는 침묵하거나 토픽 카드에 위임.

### GAP-B. 직면 능력 부재
현재 코치는 순수 응원기계 ("오직 이 사람이 얼마나 대단한가에 집중").
회피 패턴(같은 할일 3일+ 미룸)을 진단·재검증하는 능력 없음.
→ 카운슬: 잔소리 X, 무죄추정 호기심으로 "이거 진짜 네 목표 맞아?" 필요.

### GAP-C. 복귀 시나리오 전무
오랜 공백 후 복귀 감지·환대 로직 없음.
→ 카운슬: 죄책감 0, 알림 폭탄 금지, 공백을 데이터로 재프레이밍, 마지막 맥락 기억해 되묻기.

### GAP-D. 콜드 스타트가 패시브
SITUATION_1은 사용자가 카드를 **선택**해야 시작 (passive).
→ 카운슬: 코치가 열린 질문 1개로 **먼저** 연다 (proactive). 입력칸 아닌 질문으로 시작.

### GAP-E. 런타임 신호 부족
`buildContextBlock`에 시간대(밤늦음), 마지막 접속 이후 경과, 연속 완료 streak,
회피 카운트 등 **상태 신호**가 없음 → 모서리 시나리오 트리거 불가.

## 4. 미해결 제품 결정 (사용자 확정 필요)

**FORK-1 (코치의 소울)**: 현재 코치는 의도적으로 "리드 금지 + 순수 긍정"으로 설계됨.
사용자는 "리드 + 필요한 말(직면 포함)"을 원한다고 표명.
- (A) **추천** — 긍정을 *기본 영혼*으로 유지하되, 특정 순간에만 게이트된 리드/직면 추가.
  직면도 항상 긍정으로 열고, 비난이 아닌 호기심으로. 콜드 스타트에서만 리드 허용.
  → 사용자가 사랑하는 장영란/차홍 에너지(안전·고양감)를 지키면서 차원만 확장.
- (B) 코치를 더 직설적/도전적 페르소나로 재정의 (소울 교체).

→ **A 채택 시** GAP-A의 "리드 금지" 규칙을 *맥락 조건부*로 완화 (자유대화 = 안 밀어붙임, 콜드스타트/온보딩 = 리드).

## 5. 단계별 구현 계획

### Phase 1 — 프롬프트 영혼 확장 (저위험, chat.ts only)
- `COACH_SYSTEM_PROMPT`에 추가:
  - 콜드 스타트 행동: 재료 없으면 열린 질문 1개로 먼저 연다. 표면 욕망("부자") → "그럼 뭐가 달라져" 한 번 더 → 사용자 단어로 정체성 문장 응결.
  - 직면 행동(게이트): 회피 신호가 컨텍스트에 있을 때만, 무죄추정 호기심으로 재검증. 먼저 부끄러움 무장해제.
  - 복귀 행동: 공백 신호가 있을 때 죄책감 0 + 마지막 맥락 되묻기.
  - GAP-A 완화: "리드 금지"를 맥락 조건부로 수정.
- 검증: `npm run typecheck` (프롬프트는 문자열이라 타입 영향 없음, 빌드 확인)

### Phase 2 — 런타임 상태 신호 주입 (중위험, chat.ts + 클라이언트)
- `buildContextBlock`에 신호 추가: 현재 시간대, 마지막 활동 이후 경과시간, 연속완료 streak, 회피(미완료 반복) 카운트.
- 클라이언트(`CoachChat.tsx`/훅)에서 이 신호를 계산해 body에 전달.
- 검증: typecheck + dev 서버에서 각 신호가 컨텍스트 블록에 들어가는지 콘솔 확인.

### Phase 3 — 콜드 스타트 proactive 전환 (UX, CoachChat.tsx)
- 목표 0개일 때 토픽 카드 대신(또는 카드 위에) 코치의 첫 질문 자동 표시.
- 검증: dev 브라우저 QA (`?dev=1`), 신규 사용자 플로우.

### Phase 4 — coachModes.ts 정리 (리팩터)
- 죽은 코드면 제거, 살아있으면 chat.ts와 일원화.
- 검증: grep으로 참조 0 확인 후 제거 + typecheck.

### Phase 5 (보류) — 도크 카드
- `todo.md`의 코치 도크 카드 = Phase 2의 상태 신호를 트리거로 사용하는 proactive UI.
