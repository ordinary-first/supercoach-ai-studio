# /learn — 지식 기록 스킬

사용자가 `/learn`을 호출하면 `.Codex/knowledge/`에 새 지식을 기록한다.

## 실행 흐름

### 1. 기록 대상 파악

$ARGUMENTS가 있으면 내용을 분석하여 적절한 파일을 자동 선택.
$ARGUMENTS가 없으면 사용자에게 질문:

"무엇을 기록할까요?"
1. **문제 해결** (삽질 끝에 알아낸 것) → `til.md`
2. **패턴/관례** (이 프로젝트에서 이렇게 하는 이유) → `patterns.md`
3. **외부 팁** (X/웹에서 찾은 유용한 정보) → `expert-tips.md`
4. **기술 결정** (왜 A 대신 B를 선택했는지) → `decisions.md`

### 2. 기존 내용 확인

선택된 파일을 읽어서 중복 여부 확인. 이미 유사한 내용이 있으면 업데이트 제안.

### 3. 기록

각 파일의 형식에 맞춰 항목 추가:

**til.md 형식:**
```markdown
## YYYY-MM-DD: 문제 한 줄 요약
- **증상**: 무엇이 안 됐나
- **원인**: 왜 안 됐나
- **해결**: 어떻게 고쳤나
- **파일**: 관련 파일 경로
```

**expert-tips.md 형식:**
```markdown
## [카테고리] 팁 제목
- **출처**: @username (링크)
- **핵심**: 한 줄 요약
- **적용**: 이 프로젝트에서 어떻게 쓰는지
```

**patterns.md 형식:**
```markdown
## 패턴 이름
- 설명 (2-3줄)
- 관련 파일 경로
```

**decisions.md 형식:**
```markdown
## YYYY-MM-DD: 결정 제목
- **결정**: 무엇을 결정했나
- **이유**: 왜 그렇게 결정했나
- **트레이드오프**: 뭘 포기했나
- **영향**: 어떤 파일/구조에 영향
```

### 4. 확인

기록 완료 후 사용자에게 한 줄 요약 출력:
"knowledge/{파일}에 기록 완료: {제목}"
